GeoTIFF Product IO Specification

Requirement Comment
The writer shall be able to write multiple bands. Current: One image(IFD) with multiple bands, because
GeoTIFF files with multiple images not supported by ENVI, ArcView.
It's required that the written output can be read by the reader. Current: Fulfilled
The IO module shall support the old push and the new pull processing model.
Honor the ProductWriter interface. 		Current:
JAI and IIO are not able to write band data separately.
Both can only write multiple images separately but not the bands of an image. Multiple images are not applicable because common GIS tools do not support this format.

We can use the offset given by the IFD and write the data directly to the stream.
It shall be possible to read and write super-sized images (pixel count > 231-1). With JAI possible.
With IIO 1.1 it does not work, due to a bug (BUG-104). With 1.2 (Daily-Build) it works.
Color maps (Index Coding) shall be supported (ClobCover). Only limited by current BEAM capabilities (ColorManipulationView, Stx).
Virtual bands shall be written (write the data). Easy to write the data (when read no VB any more).
To write them as real VBs all related things to BandArithmetic have to be saved (referenced tie-points, flags, ...).
All geo coded products shall have a geo-coding when written. Current: At least tie-points are written for all not directly supported geo-codings.
All projections supported by BEAM shall be supported by the Reader/Writer. Current:
All MapProjections are supported.
When read only WGS_84 and WGS_72 datum are supported.
For the affine transformation (MapInfo) shearing shall be supported (Nest). Still missing.
BEAM Metadata shall be stored in GeoTIFF. Currently supported:
  • product name
  • product type
  • band name
  • description
  • unit
  • data type
  • no-data value, no-data value used
  • scaling factor, scaling offset, log10 scaled
Tiling, subsampling (multi resolution levels) shall be supported. Tiling is supported but not subsampling.
The raw data values shall be written. Current: Fulfilled
In BEAM the raw data shall be scaled to geophysical values. Current: Fulfilled
Requirements by major tools. Major Tools:
ENVI --> One IFD with multiple Bands
ArcView --> One IFD with multiple Bands
RGB image as Input Is converted to 3 separate bands.

TIFF/GeoTIFF Features

Feature Support
Multiple images within an file. Not supported
Image pyramids. Not supported
Multiple bands in one image Supported
Color maps Supported
Projected Coordniate Systems Only those supported by BEAM. They have to be on the geodetic datum WGS_84 and WGS_72.
Geographic Coordniate Systems They have to be on the geodetic datum WGS_84 or WGS_72.
Compression Images are written uncompressed. When reading images the compression is considered. Compression may slow down reading.
