About BEAM Version 4.1

Introduction

The BEAM software is a set of executable tools and an application programming interface (API) facilitating the use of the data products of a number of earth observation satellites. BEAM has initially been developed to support the data of the MERIS, AATSR and ASAR sensors onboard the ESA Envisat platform. BEAM is developed under contract to ESA/ESRIN.

Directory Structure

This is the structure of the BEAM installation directory:

  • bin - BEAM/VISAT executable and BEAM command-line tools.
  • config - BEAM configuration file.
  • lib - Common libraries used by BEAM.
  • modules - Installed BEAM modules and plugins.
  • licenses - 3rd party licenses and EULAs of libraries.

If BEAM is started the first time, a .beam directory is created in the user's home directory. BEAM uses this directory to store user preferences, auxiliary and temporary data.

Third Party Content

BEAM includes some items that have been sourced from third parties. The licenses for these items are provided in the licenses directory (see above). Especially an open-source license of the JIDE Java and Swing Components Library has kindly been made available for the BEAM project. These are distributed under the JIDE EULA and may only be used indirectly in the context of BEAM.

Development

If you want to develop software using BEAM libraries or if you want to develop BEAM plugins (modules) you have to obtain the source distribution of BEAM.

Since BEAM is developed using the Java programming language it requires a J2SE 6 JDK installed on your system.

Furthermore BEAM 4 requires Apache Maven 2 build tool to be installed and set-up on your computer.

To start development, open a console window or a shell and cd into the directory where you have unpacked the source distribution. To build BEAM, type

    mvn package

You will notice that some dependencies to common libraries cannot be resolved at this time. You will have to manually add these dependencies to your local Maven repository. For more information on how to do this, please refer to the install:install-file goal of the Maven Install Plugin. Then run mvn package again until all dependencies are resolved. We will work on this cumbersome issue in upcoming releases of BEAM.

To install the BEAM modules into your local Maven repository, type

    mvn clean install

To create the project files for your IDE, type

    mvn idea:idea
for IntelliJ IDEA or 
    mvn eclipse:eclipse
for the Eclipse IDE.

To create the final BEAM distribution archive and the installer files, type

    mvn package assembly:assembly
