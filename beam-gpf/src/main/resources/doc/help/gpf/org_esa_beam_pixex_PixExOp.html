PixEx Operator

PixEx Operator Description

Overview

Name:PixEx
Full name:org.esa.beam.pixex.PixExOp
Purpose:Extracts pixels from given locations and source products.
Version:1.0

Description

This operator is used to extracts pixels from given locations and source products. It can also create sub-scenes containing all locations found in the source products and create KMZ files which contain the locations found in a source product

Sources

Name Description
sourceProducts

Parameters

Name Type Description Default
inputPaths File[] The paths to be scanned for input products. May point to a single file or a directory. If path ends with '**' the directory is scanned recursively.
exportBands Boolean Specifies if bands are to be exported true
exportTiePoints Boolean Specifies if tie-points are to be exported true
exportMasks Boolean Specifies if masks are to be exported true
coordinates Coordinate[] The geo-coordinates
timeDifference String The acceptable time difference compared to the time given for a coordinate. The format is a number followed by (D)ay, (H)our or (M)inute. 1D
coordinatesFile File Path to a file containing geo-coordinates. BEAM's placemark files can be used.
windowSize Integer Side length of surrounding window (uneven) 1
outputDir File The output directory.
outputFilePrefix String The prefix is used to name the output files. pixEx
expression String Band maths expression (optional). Defines valid pixels.
exportExpressionResult Boolean If true, the expression result is exported per pixel, otherwise the expression is used as filter (all pixels in given window must be valid). true
exportSubScenes boolean If set to true, sub-scenes of the regions, where pixels are found, are exported. false
subSceneBorderSize int An additional border around the region where pixels are found. 0
exportKmz boolean If set to true, a Google KMZ file will be created, which contains the coordinates where pixels are found. false
